# AI Statement and Essay

1. I used generative AI on this assignment to help understand the requirements, plan the stack class, set up the correct files, write the stack methods, and create tests for the stack. I also used AI for Git help because I had problems with the repository setup, duplicate README files, and making sure my files were actually pushed to the GitHub Classroom repo. I used AI because this assignment required AI usage, but I still had to run the commands, check the output, fix problems, and make sure the final project matched the assignment instructions.

2. One error AI made was assuming I should keep the uppercase `README.md` file because that is the standard name and the one the GitHub Classroom repo originally gave me. That made sense, but I noticed that the lowercase `readme.md` was the one I had actually added my README text to. I corrected the issue by deciding which README file I wanted to keep and removing the duplicate. Another issue was that I tried to use `git rm README.md -m "removed duplicate readme"`, but `-m` does not work with `git rm`. I learned that the remove command and the commit message are separate steps.

3. One instance where AI code was correct but I could improve it was the testing setup. The basic stack code and test structure made sense, but I had to make sure the testing matched the assignment requirements and was not just a few simple examples. I checked that the tests covered empty stack behavior, pushing, popping, peeking, underflow, overflow, null pointer handling, LIFO order, and random operations. I also made sure the tests used `STACK_SIZE` instead of hardcoded numbers so the tests would still work if the stack size changed.

4. One thing I learned by using AI was how to remove files from a Git repository through the command console. Before this assignment, I did not know the correct way to remove a tracked file using commands. I learned that `git rm filename` removes the file from the repo, and then I still need to commit that change with `git commit -m "message"`. I also learned that if a file is just an untracked extra file, like a compiled `a.out`, I can remove it with `rm a.out` and not commit it if Git was never tracking it.

5. I do not think I would have done better without AI on this assignment. I still had to understand the stack ADT, run the code, compile it, test it, fix Git issues, and make sure the repository matched the submission rules. AI helped me move through problems faster, especially with Git and testing, but it did not replace checking the assignment requirements myself. Without AI, I probably would have spent a lot more time stuck on repository issues and may have submitted extra files or missed part of the testing requirements.
